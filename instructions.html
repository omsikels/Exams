System Setup & Installation

Prerequisites

  • Node.js (version 16 or higher)
  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • Webcam and microphone access

Step 1: Install Dependencies

npm install

Step 2: Start the Server

npm start

The server will start on http://localhost:3000

Step 3: Access the Website

  • Student Exam: http://localhost:3000
  • Admin Panel: http://localhost:3000/admin.html
  • Admin Login: http://localhost:3000/login.html

Default Admin Credentials

Username: admin

Password: exam3000

Student Guide

Taking an Exam

Step 1: Start the Exam

  • Navigate to the main website (http://localhost:3000)
  • Enter your full name in the "Your Name" field
  • Click "Start Exam"

Step 2: Camera Permission

  • Allow camera and microphone access when prompted
  • Your video will be recorded during each question
  • Ensure good lighting and clear camera view

Step 3: Answer Questions

  • Read each question carefully
  • Each question has a time limit (displayed at the top)
  • Click on your chosen answer (1, 2, 3, or 4)
  • The timer shows remaining time for the current question
  • Questions advance automatically when time expires

Important Notes for Students

  • Once you select an answer, you cannot go back
  • Video recording starts automatically for each question
  • If time runs out, the question is marked as incorrect
  • Stay within camera view throughout the exam
  • Ensure stable internet connection

Technical Requirements

  • Working webcam and microphone
  • Modern web browser with WebRTC support
  • Stable internet connection
  • Allow browser permissions for camera/microphone

After Completion

  • A completion screen will show your exam summary
  • All video recordings are automatically uploaded
  • Contact the administrator for results

Administrator Guide

Accessing Admin Panel

Step 1: Login

  • Go to http://localhost:3000/login.html
  • Enter admin credentials (admin/exam3000)
  • Click "Login" to access the admin panel

Question Management

Creating Questions

  • Use the "Questions" section in the admin panel
  • Click "Add Question" to create new questions
  • Fill in the question text
  • Add 4 multiple choice options
  • Select the correct answer by clicking the radio button
  • Set time limit for each question (in seconds)
  • Click "Save All" to save changes

Editing Questions

  • Navigate between questions using Q1, Q2, Q3... buttons
  • Modify question text and choices as needed
  • Adjust time limits for individual questions
  • Remove questions using the "Remove Question" button
  • Always save changes before switching questions

Video Management

Viewing Student Videos

  • Videos are organized by student name in the left panel
  • Click on a student's name to expand their video list
  • Videos are named: "right[#].mp4" for correct answers, "wrong[#].mp4" for incorrect
  • Click on any video to preview it in the video player

Video Segmentation (Advanced Feature)

  • Load a video in the player
  • Use "Split Segment" to mark emotion changes
  • Assign emotions (Surprise, Fear, Happiness, Sadness, Anger, Neutral) to each segment
  • Use "Generate JSON" to create emotion data
  • Click "Save" to store the emotion analysis

Question Features

  • Multiple choice (4 options)
  • Individual time limits
  • Question navigation
  • Bulk save/load
  • JSON export

Video Features

  • Automatic recording
  • User organization
  • Video preview
  • Emotion analysis
  • Segment marking

Admin Features

  • Secure login system
  • Question management
  • Video review tools
  • Data export
  • Session management

Data Management

Question Data

  • Questions are stored in "questions.json"
  • Total exam time is calculated automatically
  • Use "Generate JSON" to export question data
  • Backup question files regularly

Video Storage

  • Videos are stored in "videos/[student-name]/" folders
  • Emotion data is saved in "extracted/emotions.json"
  • Video files are in WebM format
  • Ensure sufficient disk space for video storage

Security Notes

  • Change default admin password in production
  • Restrict access to the admin panel
  • Regularly backup question and video data
  • Monitor disk space usage

Troubleshooting

Common Student Issues

Camera/Microphone Not Working

  • Check browser permissions for camera/microphone
  • Ensure no other applications are using the camera
  • Try refreshing the page and allowing permissions again
  • Test with a different browser

Video Not Recording

  • Check internet connection stability
  • Verify camera permissions are granted
  • Clear browser cache and cookies
  • Disable browser extensions that might interfere

Exam Won't Start

  • Ensure you've entered your name
  • Check that questions are configured by admin
  • Verify server is running
  • Try accessing from the correct URL

Common Admin Issues

Cannot Access Admin Panel

  • Verify you're using the correct login credentials
  • Check that the server is running
  • Clear browser cache and try again
  • Ensure you're accessing /login.html first

Videos Not Showing

  • Check that students have completed exams
  • Verify video files exist in the videos folder
  • Click "Refresh Video List" button
  • Check server console for error messages

Questions Not Saving

  • Ensure write permissions for the questions.json file
  • Check available disk space
  • Verify network connection
  • Look for error messages in browser console

Technical Information

System Architecture

  • Backend: Node.js with Express framework
  • Frontend: HTML, CSS, JavaScript (ES6 modules)
  • Video Recording: WebRTC MediaRecorder API
  • File Storage: Local filesystem with organized folder structure
  • Session Management: Express sessions with memory store

File Structure

project/ ├── server.js # Main server file ├── package.json # Dependencies ├── questions.json # Question database ├── public/ # Static web files │ ├── index.html # Student exam page │ ├── admin.html # Admin panel │ ├── login.html # Admin login │ ├── style.css # Styling │ └── js/ # JavaScript modules ├── videos/ # Student video recordings │ └── [student-name]/# Individual student folders └── extracted/ # Emotion analysis data └── emotions.json # Emotion segments data

API Endpoints

  • GET /api/questions - Fetch exam questions
  • POST /api/questions - Save questions (admin)
  • POST /api/upload-video - Upload student video
  • GET /api/videos - List all videos (admin)
  • POST /api/save-emotions - Save emotion analysis
  • POST /login - Admin authentication
  • POST /logout - End admin session

Browser Compatibility

  • Chrome 60+ (Recommended)
  • Firefox 55+
  • Safari 11+
  • Edge 79+
  • Requires WebRTC support for video recording

Performance Notes

  • Video files are stored locally, ensure adequate disk space
  • Large video files may affect upload performance
  • Consider implementing video compression for production use
  • Monitor server resources with multiple concurrent users

Production Deployment

  • Change default admin credentials
  • Use HTTPS in production environment
  • Configure proper file permissions
  • Set up regular data backups
  • Monitor disk space and server performance
  • Consider using a database for question storage
